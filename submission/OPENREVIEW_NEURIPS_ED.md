# NeurIPS 2026 Evaluations & Datasets Track — submission bundle

## Hosting (required before desk rejection)

| Artifact | Where | Status |
|----------|--------|--------|
| Dataset | **Hugging Face** `datasets/dhruvsuri17/warp-opf-case118-dual` (or your anonymised mirror) | Run `bash scripts/neurips_publish_all.sh` after `export HF_TOKEN=...` |
| Dataset (optional duplicate) | **OpenML** | `python data/upload_to_openml.py ...` + note ID in paper |
| Code + Croissant | **Supplementary zip** on OpenReview | Zip `warp-benchmark/` minus `.git` / caches |

**Double-blind:** Hosting under a named Hub account (`dhruvsuri17`) can compromise anonymity. For strict double-blind review, use an anonymous HF org/account or a private dataset repo with reviewer access, then migrate to your named Hub account at camera-ready. If that is impossible, request **single-blind** in the submission form per NeurIPS E&D guidelines.

## Croissant (required)

- Root file: **`croissant.json`** (generated by `scripts/build_release_tarballs_and_croissant.py`).
- Includes **core** Croissant 1.0 fields + **RAI** (`rai:*`) per track rules.
- Validate locally: `python scripts/validate_croissant.py`
- Online: [ML Commons Croissant validator](https://huggingface.co/spaces/MLCommons/croissant-validator)

After **OpenML** upload, regenerate Croissant with `--openml-id <id>` or run `python scripts/update_croissant.py` if you maintain parallel OpenML URLs.

## Large datasets (> 4 GB)

Your Case118 dual tarballs are ~27 MB compressed — **no reviewer sample required** by size. Optional tiny peek: `submission/reviewer_sample/` (if populated).
